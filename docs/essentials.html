Cowork Essentials

The fast-paced track. The instructor walks you through M01 (full install) live, then guides you through the rest. By the end you have a personalized AI workspace, connectors to your real data, a chief-of-staff brief, 7 custom skills, and the Daily Command Center. 12 essential moves, each one direct: what to do, what prompt to try, what comes next. Whenever you want a deeper dive on any module, type /teach <module> in Cowork and the tutor walks you through it patiently. Or jump straight to the full course.

1. Welcome + full install (M01)

The live install walkthrough. The instructor narrates each step on screen; you click along on your own machine. By the end you have everything the rest of the modules need. Anything that fails — flag the instructor, don't move on.

1.1 Install Claude Desktop with Cowork

  1. Download Claude Desktop from claude.com/download.
  2. Run the installer.
  3. Open Claude Desktop and sign in with your Anthropic Max account.
  4. Switch to the Cowork tab in the left sidebar.

Verify: the Cowork sidebar is visible with sections for Customize, Plugins, and Cowork Projects.

1.2 Install Obsidian

  1. Download Obsidian from obsidian.md/download.
  2. Run the installer.
  3. Open Obsidian and point it at your vault folder (in your cloud-sync app — Dropbox / OneDrive / Drive / iCloud Drive).

Verify: your vault loads in Obsidian. You see folders like Daily Notes/, References/, about-me/.

1.3 Install Chrome + the Claude browser extension

  1. Download Chrome from google.com/chrome if you don't already have it (Brave or Edge work too).
  2. Open the Chrome Web Store: chromewebstore.google.com.
  3. Search Claude (official, by Anthropic). Click Add to Chrome.
  4. Pin the extension via the toolbar puzzle-piece icon → pin Claude.
  5. Click the Claude icon and sign in with the same Anthropic account.

Verify: clicking the Claude icon in Chrome opens the side panel.

1.4 Install the ExecAIDay course plugin (in Cowork)

  1. Open the Customize sidebar.
  2. Add the marketplace first. Click + to add a personal marketplace. Paste: github.com/The-AI-Guys-BV/ExecAIDay-course. Confirm. Cowork syncs.
  3. Then install the plugin. Go to Personal plugins (not the Anthropic plugins section). Click Install on execaiday.

Verify: type /teach in Cowork — expect a list of 16 modules. If not, the plugin install didn't complete.

1.5 Authenticate your connectors

  1. In Cowork, go to Customize → Connectors.
  2. Click + and add each one your work depends on: Gmail or Microsoft 365, Calendar, Drive, Slack, plus your custom apps (Asana, Jira, Linear, SFDC, HubSpot — whatever you use).
  3. OAuth approve each one in your browser.

Verify: all expected connectors show Connected in the Customize → Connectors panel.

1.6 Install the Claude for Word, Excel, PowerPoint add-ins

This adds a Claude sidebar inside each Office app. Conversation context is shared across all three — Excel analysis flows into a PowerPoint deck flows into a Word memo without copy/paste. Requires a paid Claude plan (Pro / Max / Team / Enterprise).

  1. Open Word. Go to the Insert tab → Get Add-ins (or Office Add-ins, depending on your version).
  2. AppSource opens in a dialog. Search Claude by Anthropic. The official add-in is published by Anthropic.
  3. Click Add. Approve the prompts. The Claude sidebar appears in Word.
  4. Sign in with the same Anthropic account you use for Cowork.
  5. Repeat for Excel and PowerPoint. Open each app, same flow: Insert → Get Add-ins → search "Claude by Anthropic" → Add → sign in.

Verify: open Word, Excel, and PowerPoint — the Claude icon is in the ribbon of all three. Click it; the Claude sidebar opens.

1.7 Confirm vault + work folder are in a cloud-synced location

Your vault and work folder must live inside a cloud-sync app (Dropbox / OneDrive / Google Drive / iCloud Drive). This is required, not optional — it's how the vault stays backed up, accessible from your phone for Dispatch (step 11), and durable across machines.

Verify: navigate to the folder paths in Finder / Explorer — both should sit inside your cloud sync folder.

1.8 Create your Cowork Project and mount both folders

  1. In the Cowork sidebar, click + New Project. Name it "Workshop" (or anything).
  2. Mount your Obsidian vault (knowledge folder).
  3. Mount your work folder (projects parent).
  4. Both inside the same Cowork Project — load-bearing.

Verify: in the new Cowork Project, type "What folders am I currently mounting?" — Claude should list the vault and the work folder by name.

You now have a working AI workspace. Move to step 2.

8. Folders, projects, and the two files Claude reads automatically (M09)

Two files shape how Claude behaves in any folder:

  • claude.md — per-folder context. Claude reads this first. The closer to the file Claude is editing, the higher priority it wins.
  • memory.md — patterns Cowork learns over time. Claude appends; you can edit by hand.

To scaffold a new project — bounded work with Source Materials / Working Files / Deliverables, plus a tailored claude.md:

/new-project board-deck-q3

Claude interviews you (outcome, stakeholders, deadline, tone), creates the project folder, writes a claude.md that inherits your CoS brief plus project specifics. Every conversation in that folder gets project context for free.

Verify: Check your work folder — expect the new project subfolder with source/working/deliverables structure.

10. Make a branded PowerPoint (M11)

The Word, Excel, PowerPoint Claude add-ins are installed and signed in. The branded-deck pattern is two stages: extract the branding once and save it, then reuse it for every new deck.

Stage 1 — Extract branding from a reference deck (one-time)

  1. Start a new Cowork chat (or a new Cowork Project called something like "Branding extraction") so the conversation context stays clean and isolated from other work.
  2. Mount the relevant vault folder. In the Cowork Project, mount <vault>/References/ (or just <vault>/References/brand/ if you want a tighter scope). This is where the saved branding will live.
  3. Add your reference PowerPoint to the project. Pick a recent on-brand deck (board update, investor pitch, strategy review — any topic, only the styling matters) and either:
    • Drop the file into the mounted folder (e.g., copy it to <vault>/References/brand/source-deck.pptx), or
    • Mount the folder where the deck already lives.
  4. Ask Claude to extract and save the branding — paste this in the Cowork chat: 
    Read <vault>/References/brand/source-deck.pptx (or whatever the file
is called). Extract the visual style:
- Brand colors (with hex codes)
- Fonts for headings and body
- Logo placement (which corner, what size)
- Slide layouts (title slide, content slide, section divider)
- Header / footer treatment

Save the spec as <vault>/References/brand/branding.md.
Save the logo image as <vault>/References/brand/logo.png.
Confirm both files when written.

Verify: open <vault>/References/brand/branding.md in Obsidian — expect a clean spec listing colors, fonts, layouts; logo.png sits next to it.

You now have a reusable brand spec. Do this once.

Stage 2 — Build a new deck using saved branding

  1. Start a new Cowork Project for the deck — e.g., /new-project q3-board-deck. Project folder gets created in your work folder.
  2. Mount the branding folder in the new project — add <vault>/References/brand/ to the project's mounted folders. Now Claude in this project sees both the project's working files AND the branding spec.
  3. Open a blank PowerPoint in the project's Deliverables folder.
  4. In the PowerPoint Claude sidebar, ask: 
    Build me a 5-slide deck on [your real topic — e.g., "Q3 board update on
revenue trajectory"]. Use the branding spec at
<vault>/References/brand/branding.md and the logo at
<vault>/References/brand/logo.png. One bold takeaway per slide,
one supporting data point, no clutter.

Verify: the deck Claude generates uses your brand colors, fonts, and logo placement — recognizably yours, not Claude's defaults.

Bonus — Cross-app chain in the same project

Inside the same Cowork Project, the Excel, PowerPoint, and Word add-ins share context. So you can:

  1. Open your Q3 financial model in Excel inside the project. Ask Excel-Claude to analyze and surface top trends and risks (writes comments into the relevant cells).
  2. Switch to the PowerPoint deck in the same project. Ask PowerPoint-Claude to "build a slide section based on the Excel analysis we just did." It remembers — no re-explaining.
  3. Switch to a Word memo. Ask it to "draft a 1-page memo summarizing the Excel + PPT work for the board chair." It pulls the same context.

The conversation persists across the three apps because they're all attached to the same Cowork Project.

11. Schedule Claude to work while you sleep (M12)

In Cowork: Settings → Scheduled tasks → New. Pick a cadence (e.g., Mondays at 8:30am), give it a prompt, send the output to a Slack DM, an email draft, or a vault file. Example:

Tell me what's important this week. Pull from my calendar (next 7 days),
my recent emails (last 48h), and my Slack mentions. Format as a 5-line
brief. Send as a Slack DM to me.

12. Build your Daily Command Center (M14)

The flagship artifact. One live page that pulls calendar, urgent email, drafts waiting, Slack mentions, and yesterday's reflection — all in one view that refreshes every time you open it. In Cowork:

Build me a live artifact called "Daily Command Center" with these tiles:
1. Today's calendar (Calendar connector).
2. Top 3 urgent emails (Gmail / M365).
3. Drafts waiting for me to send.
4. Slack mentions overnight.
5. Yesterday's daily note closing paragraph.

Layout 2 columns. Refresh on every open.

Open this every morning. Done.

Verify: Reopen the artifact multiple times — expect fresh data each time.
