Cowork Essentials

The fast-paced track. Doc Tony handles all setup in advance — Cowork, Obsidian, Chrome + the Claude extension, the ExecAIDay plugin (with /teach and the rest of the skills), every connector, the Office add-ins, your starter folders. You walk in to a working system and walk out using it. 12 moves, each one direct: what to do, what prompt to try, what comes next. Whenever you want a deeper dive on any module, type /teach <module> in Cowork and the plugin walks you through it patiently. Or jump straight to the full course.

1. Confirm everything is alive (M01)

Open Claude Desktop, switch to the Cowork tab. Find your workshop Cowork Project — your vault and work folder are mounted. Type /teach in the chat. You should see a list of 16 modules. That's your safety net: /teach <module> walks any topic patiently, on demand.

Deep dive: type /teach M01 in Cowork, or open the full M01 page.

2. Sharpen any prompt with R-C-T-F-C (M02)

The best prompts have five elements: Role, Context, Task, Format, Constraints. Skip any one and the output is generic.

If you don't want to remember all five, type:

/help-me-prompt help me write a follow-up email to a candidate I want to hire

Claude takes your draft and rewrites it in R-C-T-F-C structure. Ship the rewrite. Deep dive: type /teach M02 in Cowork, or open the full M02 page.

3. Build a live artifact (M04)

An artifact is a generated output that renders inline. Live artifacts pull fresh data each time you open them. In Cowork:

Build me an interactive page showing the days of the week in a 2x4 grid,
each in a different color, with today highlighted.

Renders right there. Close it, reopen — it refreshes. The Daily Command Center (step 11) is the same idea, scaled up. Deep dive: type /teach M04 in Cowork, or open the full M04 page.

4. Operate websites with Claude in Chrome (M05)

Open Chrome (or Brave). Click the Claude extension icon — sidebar opens on the right. Open any portal you actually use (CRM, dashboard, vendor portal). In the sidebar:

Pull the next renewal date from the customer profile open in this tab.

Or record a 3-step workflow once and replay it on demand later. Or open 6 competitor sites in tabs and ask Claude to compile a comparison. Deep dive: type /teach M05 in Cowork, or open the full M05 page.

5. Build your own slash command (M07)

You've already used /teach and /help-me-prompt — those are skills (slash commands). To build your own:

/skill-creator

Claude interviews you about a task you do often (a weekly status report, a specific kind of email follow-up, a pre-1:1 brief), drafts the skill, writes it to your machine. Then it's a slash command you can invoke forever. The autonomy moment. Deep dive: type /teach M07 in Cowork, or open the full M07 page.

6. Write your chief-of-staff brief (M06)

The highest-leverage file in the workshop. Once it exists, every conversation inherits your voice, priorities, stakeholders, and tone. In Cowork:

Walk me through writing my chief-of-staff brief. Use the new-vault-claudemd
pattern. Save the result to <vault>/About Me.md (create it if it doesn't
exist). Then make sure <vault>/claude.md contains the one-line pointer:
"Chief-of-staff brief: read About Me.md."

Claude interviews you for a few minutes — role, voice, stakeholders, what you avoid. Output is a 30-line brief that personalizes everything Claude does for you afterward. Deep dive: type /teach M06 in Cowork, or open the full M06 page.

7. Use Claude to update your Obsidian vault (M08)

Your vault is mounted in your Cowork Project. That means Claude reads every note and writes back to any of them. Try:

Write a 5-line journal entry about how the workshop is going so far,
in journal voice. Save to today's daily note in my vault.

Open Obsidian — the note is there, rendered. Same workflow scales: summarize references, tighten existing notes, expand outlines. Deep dive: type /teach M08 in Cowork, or open the full M08 page.

8. Folders, projects, and the two files Claude reads automatically (M09)

Two files appear in every folder of your workspace and shape how Claude behaves there:

  • claude.md — per-folder context. Claude reads this first when working in or under that folder. The closer to the file Claude is editing, the higher priority the claude.md wins. Vault-root claude.md applies everywhere; a project-folder claude.md overrides for files inside that project.
  • memory.md — patterns Cowork learns about that folder over time. Claude appends to it as you work; you can edit by hand.

To scaffold a new project — bounded body of work with Source Materials / Working Files / Deliverables buckets and a tailored claude.md — type:

/new-project board-deck-q3

Claude interviews you (outcome, stakeholders, deadline, tone), creates the project folder under your work folder, writes a synthesized claude.md that inherits your chief-of-staff brief plus this project's specifics, and creates a matching project file. From now on every conversation that touches files in that folder gets the project context for free. Deep dive: type /teach M09 in Cowork, or open the full M09 page.

9. Verify your connectors (M10)

Doc Tony already authenticated Gmail / M365, Calendar, Drive, Slack, and your custom apps. Confirm in Customize → Connectors — all show Connected. Try one:

/email-triage today

Claude classifies your inbox into urgent / reply / read / archive / delete. Or:

/drafts-inbox new

Claude scans recent emails needing replies, drafts them all to a review surface. Deep dive: type /teach M10 in Cowork, or open the full M10 page.

10. Make a branded PowerPoint (M11)

Doc Tony installed the Word, Excel, PowerPoint Claude add-ins. Open any recent on-brand PowerPoint deck of yours plus a blank one. In the blank deck's Claude sidebar:

Open <path-to-the-reference-deck>.pptx. Extract the visual style — fonts,
color palette, logo placement, slide layouts. Then build me a 5-slide deck
on [topic] using exactly that style. Don't copy the content — only the look.

Output looks like YOUR company's deck, not Claude's defaults. Cross-app context: do an Excel analysis first, then have PowerPoint Claude pull it in. Deep dive: type /teach M11 in Cowork, or open the full M11 page.

11. Schedule Claude to work while you sleep (M12)

In Cowork: Settings → Scheduled tasks → New. Pick a cadence (e.g., Mondays at 8:30am), give it a prompt, send the output to a Slack DM, an email draft, or a file in your vault. Example:

Tell me what's important this week. Pull from my calendar (next 7 days),
my recent emails (last 48h), and my Slack mentions. Format as a 5-line
brief. Send as a Slack DM to me.

Mobile capture works the other way: phone → Dispatch → desktop Cowork → runs whatever skill. Deep dive: type /teach M12 in Cowork, or open the full M12 page.

12. Build your Daily Command Center (M14)

The flagship artifact. One live page that pulls calendar, urgent email, drafts waiting, Slack mentions, and yesterday's reflection — all in one view that refreshes every time you open it. In Cowork:

Build me a live artifact called "Daily Command Center" with these tiles:
1. Today's calendar (Calendar connector).
2. Top 3 urgent emails (Gmail / M365).
3. Drafts waiting for me to send.
4. Slack mentions overnight.
5. Yesterday's daily note closing paragraph.

Layout 2 columns. Refresh on every open.

Open this every morning. Done. Deep dive: type /teach M14 in Cowork, or open the full M14 page.
