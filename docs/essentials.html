1. Welcome + full install (M01)

The first hour. The instructor walks you through every install step live; you click along on your own machine. By the end you have a working AI workspace. None of these tools assume you've used them before — read each section's "what this is" line if a word looks unfamiliar. Anything that fails: flag the instructor, don't move on.

Vocabulary you'll meet in this hour

The five words below show up over and over. Read once now; we'll use them throughout.

  • Vault. Obsidian's word for "a folder of notes." Everything inside the vault folder is part of one vault. The vault is just a regular folder on your computer — nothing magical. You'll create yours inside a cloud-sync app (Dropbox / OneDrive / Google Drive / iCloud Drive) so the contents are backed up automatically and accessible from your phone.
  • Cowork Project. Cowork's word for "an AI workspace." A Cowork Project remembers your conversations, mounts (includes) folders, and holds its own settings. You can have several Cowork Projects — one for the workshop, others later for sensitive work. Today you're creating exactly one, called "Workshop."
  • Mount. Telling a Cowork Project "include this folder in your context." Mounting doesn't copy or move the folder — Claude just gets read/write access to it from inside that Project. Folders can be mounted in multiple Cowork Projects.
  • Connector. An authenticated bridge from Cowork to one of your services (Gmail, Calendar, Slack, etc.). Once you've signed in once via the service's normal login page, Cowork can read your inbox, query your calendar, and so on — on your behalf, with the same permissions you have.
  • Plugin / skill / slash command. A "skill" is a pre-built workflow Cowork can run. You trigger it by typing /<name> in chat — e.g., /teach, /capture. Skills come bundled in "plugins" you install once. The workshop's plugin (ExecAIDay) gives you /teach plus 25 other skills. Plugins live at a "marketplace" — for our plugin, that's a GitHub URL.

OK, here's the install sequence. Eight steps.

1.1 Install Claude Desktop with Cowork

What this is. Claude Desktop is the app where you'll do all your AI work. Inside Claude Desktop is a tab called Cowork — that's the workspace mode. (The other mode, Chat, is for one-off questions; Cowork is for everything else.)

  1. Open claude.com/download in your current browser.
  2. Click the download for your OS (Mac or Windows). The installer file lands in your Downloads folder.
  3. Double-click the installer. Drag the Claude app to Applications (Mac) or follow the Windows installer.
  4. Open Claude Desktop. Sign in with your Anthropic account (use your work email; if you don't have an account yet, click "Sign up" and follow the flow — your subscription will be a Max plan).
  5. Look at the left sidebar. You'll see two tabs: Chat and Cowork. Click Cowork.

Verify: the Cowork tab is open, with sections in the sidebar for Customize, Plugins, and Cowork Projects. If you only see Chat, your version is too old — update from claude.com/download.

1.2 Install Obsidian and create your vault

What this is. Obsidian is a free note-taking app that stores notes as plain files in a folder on your computer. That folder is your "vault." Cowork can read and write the same files; Obsidian gives you a nice UI for browsing and editing them. The two apps don't talk to each other — they just share the same files.

Where the vault lives. Inside your cloud-sync folder. So if you use Dropbox, your vault path will be something like ~/Dropbox/Workshop-Vault. Same idea for OneDrive, Google Drive, iCloud Drive. Required: the vault must be inside one of these — that's how it stays backed up and reachable from your phone later.

  1. Open obsidian.md/download. Download the installer for your OS.
  2. Run the installer. Open Obsidian.
  3. On the welcome screen, click Create new vault (NOT "Open existing vault" — you don't have one yet).
  4. In the dialog: name it "Workshop" (or whatever you like). For "Location," click Browse and navigate INTO your cloud-sync folder (Dropbox, OneDrive, etc.) before clicking Create. Example final path: ~/Dropbox/Workshop.
  5. Click Create. Obsidian opens onto an empty vault. You'll see a sidebar with a file browser (currently empty).

Verify: in Obsidian, the title bar shows your vault name. The file browser is open and empty (or shows the workshop's starter content if your assistant pre-seeded the folder).

1.3 Install a Chromium browser, then the Claude extension

What this is. The Claude in Chrome extension adds a Claude sidebar to your browser. Claude can then read any web page you have open and operate websites for you (pull data from a CRM, fill forms, click through workflows). The extension only works in Chrome, Brave, or Edge — Chromium-based browsers. If you're on Safari, you need to install one of these.

  1. If you don't already have Chrome, Brave, or Edge: download Chrome (or Brave or Edge). Run the installer.
  2. Open the browser. Sign in with your work Google or Microsoft account so bookmarks and passwords sync.
  3. Open the Chrome Web Store: chromewebstore.google.com.
  4. In the search box, type Claude. Look for the official extension by Anthropic (it'll have an Anthropic logo and verified-publisher mark).
  5. Click Add to Chrome. Confirm the permissions prompt. The extension installs in seconds.
  6. Pin the extension to your toolbar so you can find it: click the puzzle-piece icon (right of your address bar) → click the pin icon next to "Claude" → close the puzzle menu.
  7. Click the Claude icon (now in your toolbar). A side panel opens. Sign in with the same Anthropic account you used for Claude Desktop.

Verify: the Claude icon is visible in your browser's toolbar. Clicking it opens a side panel where you can chat with Claude about whatever page you're on.

1.4 Install the ExecAIDay course plugin (in Cowork)

What this is. A plugin is a bundle of skills (slash commands) Cowork can run. The ExecAIDay plugin gives you /teach (a patient tutor for any module of this course) plus 25 other skills. It's hosted on GitHub. Cowork installs it from there in two phases: first you tell Cowork "look at this GitHub URL for plugins" (that's the marketplace), then you install the actual plugin from that marketplace.

  1. In Cowork, click Customize in the left sidebar (gear icon, near the bottom).
  2. Add the marketplace first. In the Customize panel, find the Personal marketplaces section and click the + icon. A dialog opens.
  3. Paste: github.com/The-AI-Guys-BV/ExecAIDay-course
  4. Click Add (or Confirm). Cowork syncs with GitHub — takes a few seconds; you'll see "Syncing..." then a green checkmark.
  5. Then install the plugin. Go to Personal plugins (in the same Customize panel — it's a separate section from the Anthropic plugins section; make sure you're in the right one). The execaiday plugin is now in the list because of the marketplace you just added.
  6. Click Install on the execaiday plugin.

Verify: in any Cowork chat, type /teach and press Enter. You should see a list of 17 modules ("M01 — Welcome + Foundations," etc.). If you get "Unknown command," the install didn't complete — flag the instructor.

1.5 Authenticate your connectors

What this is. A connector is an authenticated bridge from Cowork to a service like Gmail, your calendar, or Slack. You sign in once via the service's normal login page; Cowork remembers and can then read your inbox, calendar, etc. on your behalf — with the same permissions your account has. No data leaves Anthropic's infrastructure; the connector just lets Claude query the service when it needs to.

  1. In Cowork, open Customize → Connectors.
  2. Click the + icon to add a new connector. A list opens of available services: Gmail, Microsoft 365 (which covers Outlook + OneDrive), Google Calendar, Microsoft Calendar, Google Drive, Slack, plus a long list of custom apps (Asana, Jira, Linear, Salesforce, HubSpot, Notion, Atlassian, etc.).
  3. Click each service you actually use. For each one: a browser window opens; sign in with your work account; click Allow to grant Cowork permission. Browser closes; the connector shows up as Connected in the Customize panel.
  4. Repeat for every service: email (Gmail or M365), calendar, file storage (Drive / OneDrive), Slack, plus any custom apps you use heavily.

Verify: in Customize → Connectors, every expected service has a green Connected tag. To test one, type in a Cowork chat: "What's on my calendar today?" — Claude should read from your calendar and answer.

1.6 Install the Claude for Word, Excel, and PowerPoint add-ins

What this is. An "add-in" is a sidebar that appears INSIDE Microsoft Word, Excel, and PowerPoint. Once installed, you'll see a Claude icon in each app's ribbon (the toolbar at the top). Click it and Claude appears as a sidebar that can read and edit the document you're working on. The killer feature: the conversation context is shared across all three apps. So you can do an analysis in Excel, then switch to PowerPoint and ask Claude to "build slides from that analysis" — it remembers. Requires a paid Claude plan (Pro / Max / Team / Enterprise).

  1. Open Microsoft Word (a blank document is fine).
  2. In the top ribbon, click the Insert tab.
  3. Look for Get Add-ins or Office Add-ins (the wording depends on your version). Click it. A dialog opens that's actually the Microsoft AppSource marketplace.
  4. In the search box, type Claude by Anthropic. Look for the official add-in published by Anthropic.
  5. Click Add. Approve the prompts that appear. The add-in installs in a few seconds.
  6. A Claude sidebar should now appear on the right of Word. If it doesn't open automatically, look for the Claude icon in the Word ribbon and click it.
  7. Sign in with the same Anthropic account you've been using for Cowork.
  8. Repeat the same flow in Excel. Open Excel → Insert → Get Add-ins → search "Claude by Anthropic" → Add → sign in.
  9. Repeat the same flow in PowerPoint. Same steps.

Verify: open Word, Excel, and PowerPoint. In each app's ribbon, the Claude icon is visible. Clicking it opens the Claude sidebar. All three apps share one conversation when you keep them open simultaneously.

1.7 Confirm vault + work folder live in a cloud-sync app

What this is. Two folders on your computer matter for the workshop: your Obsidian vault (created in step 1.2) and a work folder where projects live (your assistant has placed a starter version of this for you, or you can create one now — call it "Work" or something similar). Both must sit inside a cloud-sync app (Dropbox, OneDrive, Google Drive, or iCloud Drive). Why required:

  • Backup. If your laptop dies, your vault is in the cloud.
  • Mobile. Your phone can reach the same files (via the cloud-sync app's mobile app, or via Dispatch — covered in step 11).
  • Multi-machine. If you switch laptops, the vault follows.
  1. Open Finder (Mac) or File Explorer (Windows).
  2. Navigate to your cloud-sync folder (e.g., the Dropbox folder).
  3. Confirm both your vault folder and your work folder are inside it.
  4. If either folder is somewhere else (Documents, Desktop, etc.), move it into the cloud-sync folder. Then re-open Obsidian and re-point it at the new vault location (File → Open Vault in Obsidian).

Verify: both folders show the cloud-sync app's status icon next to their names (a green check, a sync arrow, etc., depending on the app). That confirms they're being synced.

1.8 Create your Cowork Project and mount both folders

What this is. A Cowork Project is the AI workspace you'll spend the workshop in. Mounting a folder = telling that workspace "include this folder in your context." We're creating one Cowork Project (called "Workshop") and mounting both your vault and your work folder in it. From now on, whenever you chat with Claude inside this Cowork Project, Claude can read and write files in both folders.

  1. In the Cowork left sidebar, click + New Project (button is usually at the top of the Cowork Projects list).
  2. Name it: Workshop. Click Create.
  3. The new Project opens. Look at the right sidebar (or wherever Cowork shows mounted folders for the current Project). Find the Mounted folders section.
  4. Click + Add folder. Browse to your Obsidian vault folder. Select it. Confirm.
  5. Click + Add folder again. Browse to your work folder. Select it. Confirm.
  6. Both folders should now appear in the Mounted folders list.

Verify: in this new "Workshop" Cowork Project, type in chat: "What folders am I currently mounting in this Cowork Project?" Claude should reply with the names of your vault and your work folder. If it says it can't see any folders, the mount didn't take — flag the instructor.

Done. You have a working AI workspace. From step 2 onward, you'll spend your time IN this Cowork Project.

10. Make a branded PowerPoint (M11)

The Word, Excel, PowerPoint Claude add-ins are installed and signed in. The branded-deck pattern is two stages: extract the branding once and save it, then reuse it for every new deck.

Stage 1 — Extract branding from a reference deck (one-time)

  1. Start a new Cowork chat (or a new Cowork Project called something like "Branding extraction") so the conversation context stays clean and isolated from other work.
  2. Mount the relevant vault folder. In the Cowork Project, mount <vault>/References/ (or just <vault>/References/brand/ if you want a tighter scope). This is where the saved branding will live.
  3. Add your reference PowerPoint to the project. Pick a recent on-brand deck (board update, investor pitch, strategy review — any topic, only the styling matters) and either:
    • Drop the file into the mounted folder (e.g., copy it to <vault>/References/brand/source-deck.pptx), or
    • Mount the folder where the deck already lives.
  4. Ask Claude to extract and save the branding — paste this in the Cowork chat: 
    Read <vault>/References/brand/source-deck.pptx (or whatever the file
is called). Extract the visual style:
- Brand colors (with hex codes)
- Fonts for headings and body
- Logo placement (which corner, what size)
- Slide layouts (title slide, content slide, section divider)
- Header / footer treatment

Save the spec as <vault>/References/brand/branding.md.
Save the logo image as <vault>/References/brand/logo.png.
Confirm both files when written.

Verify: open <vault>/References/brand/branding.md in Obsidian — expect a clean spec listing colors, fonts, layouts; logo.png sits next to it.

You now have a reusable brand spec. Do this once.

Stage 2 — Build a new deck using saved branding

  1. Start a new Cowork Project for the deck — e.g., /new-project q3-board-deck. Project folder gets created in your work folder.
  2. Mount the branding folder in the new project — add <vault>/References/brand/ to the project's mounted folders. Now Claude in this project sees both the project's working files AND the branding spec.
  3. Open a blank PowerPoint in the project's Deliverables folder.
  4. In the PowerPoint Claude sidebar, ask: 
    Build me a 5-slide deck on [your real topic — e.g., "Q3 board update on
revenue trajectory"]. Use the branding spec at
<vault>/References/brand/branding.md and the logo at
<vault>/References/brand/logo.png. One bold takeaway per slide,
one supporting data point, no clutter.

Verify: the deck Claude generates uses your brand colors, fonts, and logo placement — recognizably yours, not Claude's defaults.

Bonus — Cross-app chain in the same project

Inside the same Cowork Project, the Excel, PowerPoint, and Word add-ins share context. So you can:

  1. Open your Q3 financial model in Excel inside the project. Ask Excel-Claude to analyze and surface top trends and risks (writes comments into the relevant cells).
  2. Switch to the PowerPoint deck in the same project. Ask PowerPoint-Claude to "build a slide section based on the Excel analysis we just did." It remembers — no re-explaining.
  3. Switch to a Word memo. Ask it to "draft a 1-page memo summarizing the Excel + PPT work for the board chair." It pulls the same context.

The conversation persists across the three apps because they're all attached to the same Cowork Project.
