# Work folder — context

Each subfolder under this folder is a **project folder** — bounded work with a deliverable. Project folders use the bucket structure:
- `Source Materials/` — incoming inputs.
- `Working Files/` — in-progress.
- `Deliverables/` — finished outputs.

Defaults across all my projects (override per-project in each project's own `claude.md`):
- Deliverables in Markdown unless the project specifies otherwise.
- Source Materials are read-only references; don't edit them in place.
- Working Files is where iteration happens.

The `/new-project` skill scaffolds new project folders and synthesizes a project-specific brief.
